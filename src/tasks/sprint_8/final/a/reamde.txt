-- ПРИНЦИП РАБОТЫ --
Алгоритм находит наибольший общий префикс для нескольких запакованных строк. Сначала он распаковывает первую строку полностью, а для последующих строк распаковывает только ту часть, которая не превышает длину текущего общего префификса. На каждом шаге алгоритм сравнивает текущий общий префикс с очередной распакованной строкой и обновляет префикс как их общее начало. Процесс продолжается до обработки всех строк или пока общий префикс не станет пустым.

-- ДОКАЗАТЕЛЬСТВО КОРРЕКТНОСТИ --
Алгоритм корректен, потому что:
1) Общий префикс всех строк должен быть префиксом каждой отдельной строки
2) На каждом шаге алгоритм сохраняет только ту часть строки, которая является общим началом для всех обработанных строк
3) Ограничение длины распаковки последующих строк оптимизирует производительность без потери корректности
4) Если на каком-то шаге общий префикс становится пустым, дальнейшая обработка не нужна

-- ВРЕМЕННАЯ СЛОЖНОСТЬ --
- k - количество строк
- L - длина самой длинной распакованной строки
В худшем случае: O(k * L), где каждая строка распаковывается полностью
В лучшем случае: O(L + k * P), где P - длина общего префикса
Итоговая временная сложность: O(k * L)

-- ПРОСТРАНСТВЕННАЯ СЛОЖНОСТЬ --
- L - длина самой длинной распакованной строки (максимальный возможный размер общего префикса)
- S - максимальная глубина вложенности квадратных скобок в запакованных строках
O(L) - для хранения распакованных строк и общего префикса
O(S) - для стека в функции распаковки, где S - глубина вложенности скобок
Итоговая пространственная сложность: O(L + S)
