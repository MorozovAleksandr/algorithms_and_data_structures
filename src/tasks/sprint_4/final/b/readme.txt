-- ПРИНЦИП РАБОТЫ --
Программа реализует хеш-таблицу с методами put, get и delete для обработки запросов. Основные компоненты:

1. Хеш-таблица - использует метод цепочек для разрешения коллизий
2. Хеш-функция - вычисляет индекс как остаток от деления ключа на размер таблицы(т.к. ключи только числа)

Для каждого запроса программа выполняет операцию (put/get/delete) на хеш-таблице.
В случае ошибок (например, попытка получить или удалить несуществующий ключ) возвращается значение 'None'.

-- ДОКАЗАТЕЛЬСТВО КОРРЕКТНОСТИ --
Хеш-функция гарантирует равномерное распределение ключей по таблице.
Метод цепочек обеспечивает корректное разрешение коллизий.
Все операции (put/get/delete) работают корректно благодаря последовательному обходу цепочек

-- ВРЕМЕННАЯ СЛОЖНОСТЬ --
- n - количество запросов

put: O(1) в среднем случае
get: O(1) в среднем случае
delete: O(1) в среднем случае

Итоговая временная сложность программы: O(n)

-- ПРОСТРАНСТВЕННАЯ СЛОЖНОСТЬ --
- n - количество запросов

В худшем случае пространственная сложность Хеш-таблицы O(n)

Итоговая пространственная сложность программы: O(n)
