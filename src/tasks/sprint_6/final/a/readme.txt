-- ПРИНЦИП РАБОТЫ --
Алгоритм строит максимальное остовное дерево неориентированного графа на основе алгоритма Прима.
Он добавляет в остов вершины, выбирая на каждом шаге ребро с максимальным весом, которое соединяет уже добавленные вершины с ещё не добавленными.

Основные этапы работы:
1) Создаётся список смежности графа, в котором для каждой вершины хранится список её рёбер с весами.
2) Алгоритм начинает с вершины 1.
3) Все рёбра из текущей вершины добавляются в max-heap, где приоритет определяется весом ребра.
4) На каждом шаге из кучи извлекается ребро с наибольшим весом. Если оно соединяет уже добавленную вершину с ещё не добавленной — это ребро включается в остов.
5) Новая вершина добавляется в множество обработанных, а все исходящие из неё рёбра помещаются в кучу.
6) Процесс повторяется, пока все вершины не будут добавлены в остов, либо пока не закончатся рёбра.

-- ДОКАЗАТЕЛЬСТВО КОРРЕКТНОСТИ --
Корректность алгоритма следует из свойств алгоритма Прима, который гарантирует построение остовного дерева максимального веса при выборе рёбер с максимальными весами:
- Использование max-heap обеспечивает выбор на каждом шаге самого "тяжёлого" возможного ребра, которое не создаёт цикл.
- Каждое добавляемое ребро соединяет новую вершину с уже построенным остовом, сохраняя связность и ацикличность.
- При завершении работы все вершины, достижимые из стартовой, включены в остов, что гарантирует корректное построение максимального остовного дерева.

-- ВРЕМЕННАЯ СЛОЖНОСТЬ --
- n — количество вершин
- m — количество рёбер
Добавление всех рёбер в список смежности: O(m)
Каждая операция добавления или извлечения из кучи: O(log m)
Всего таких операций порядка: O(m)

Итоговая временная сложность: O(m log m)

-- ПРОСТРАНСТВЕННАЯ СЛОЖНОСТЬ --
- n — количество вершин
- m — количество рёбер
Список смежности хранит все рёбра: O(m)
Куча может содержать до O(m) элементов
Множества добавленных и необработанных вершин: O(n)

Итоговая пространственная сложность: O(n + m)
