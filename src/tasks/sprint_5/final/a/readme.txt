-- ПРИНЦИП РАБОТЫ --
Алгоритм использует пирамидальную сортировку (Heap Sort) для упорядочивания участников соревнования.
Основные этапы работы:
1. Создается минимальная куча (max-heap) с использованием специального компаратора для сравнения участников
2. Все элементы последовательно добавляются в кучу с поддержанием её свойств
3. Элементы извлекаются из кучи в отсортированном порядке и формируют итоговый массив

-- ДОКАЗАТЕЛЬСТВО КОРРЕКТНОСТИ --
Корректность обеспечивается свойствами двоичной кучи и правильной работой компаратора:
- При добавлении элемента выполняется операция siftUp, которая сохраняет свойства кучи
- При извлечении элемента выполняется siftDown, которая восстанавливает свойства кучи
- Компаратор гарантирует корректное сравнение участников по всем заданным критериям
- Каждый извлеченный элемент является максимальным в текущей куче, что обеспечивает правильный порядок


-- ВРЕМЕННАЯ СЛОЖНОСТЬ --
- n - количество участников

Построение кучи: O(n)
Добавление каждого элемента: O(log n) - всего O(n log n)
Извлечение каждого элемента: O(log n) - всего O(n log n)

Итоговая временная сложность: O(n log n)


-- ПРОСТРАНСТВЕННАЯ СЛОЖНОСТЬ --
- n - количество участников

Хранение кучи: O(n)
Входные данные: O(n)
Выходные данные: O(n)

Итоговая пространственная сложность: O(n)
